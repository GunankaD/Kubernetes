26. KUBECTL COMMANDS

> doskey k=kubectl $*

1. minikube start --driver=docker

2. kubectl get deployment
3. kubectl get replicaset
4. kubectl get nodes
5. kubectl get pod

CREATE DEPLOYMENTS
this is used to create pods of same type. if we need pods of different type, then we need to create different deployments
6. kubectl create deployment <name-of-deployment> --image=<base-image-for-containers>
ex: kubectl create deployment nginx-depl --image=nginx
ex: kubectl create deployment mongo-depl --image=mongo

this below command creates more replicas of the same pod type
kubectl scale deployment nginx-depl --replicas=3

EDIT DEPLOYMENTS
>choco install vim
>choco install nano
>set KUBE_EDITOR=nano
7. kubectl edit deployment nginx-depl

LOGS AND INFORMATION
8. kubectl logs <pod-id> (which can be found using >kubectl get pods)
9. kubectl describe pod <pod-id>

TO OPEN THE CMD OF ANY CONTAINER
10. kubectl exec -it <pod-id> -- bin/bash

DELETE
11. kubectl delete pods <pod-id>
12. kubectl delete replicaset <replicaset-id>
13. kubectl delete deployment <deployment-name>
- remember, doing 11 or 12 will delete the pod for sure but another would spawn in its place until and unless u delete the deployment as a whole

CREATE DEPLOYMENTS USING CONFIG FILES
14. kubectl apply -f <filename>

EXTRA INFORMATION 
15. kubectl get pods -o wide 
16. kubectl get deployment nginx-depl -o yaml

DELETE DEPLOYMENTS USING CONFIG FILES
17. kubectl delete -f nginx-deployment.yaml

INSTALL INGRESS CONTROLLER IN MINIKUBE
18. minikube addons enable ingress
19. k get pods -n ingress-nginx
- the ingress-nginx pods are created in the ingress-nginx namespace 


TO ENABLE DASHBOARD ADDON AND START IT
20. minikube addons enable dashboard
21. minikube dashboard
